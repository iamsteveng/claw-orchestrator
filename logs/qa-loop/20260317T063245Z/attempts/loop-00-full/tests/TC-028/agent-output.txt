<learnings>
- **Fake timers + real Prisma = timeout**: `vi.useFakeTimers()` with `vi.runAllTimersAsync()` or `vi.advanceTimersByTimeAsync()` both hang (60s+) when used with real Prisma in integration tests. Root cause: Prisma's multi-level async promise chains require multiple event loop iterations to resolve, but the fake timer system's internal microtask flushing loop can't drive them to completion.
- **Unit tests work, integration tests don't**: The health-monitor unit tests use mock Prisma (`vi.fn().mockResolvedValue({})`) which creates single-level promises that resolve in one microtask — hence fake timers work there. The fix for integration tests is to bypass fake timers entirely.
- **Pattern for sleep-heavy integration tests**: Directly manipulate DB state and call mocked functions sequentially. Test the state transitions (the contract) rather than the timer-driven triggering (tested adequately in unit tests).
- **monitorTenantHealth's notifyUser callback is the Slack DM mechanism**: It takes a `Notifier = (userId, message) => Promise<void>` function. In production it would call the Slack API; in tests, use a spy and assert it was called with the expected arguments.
- **pollUntilHealthy mock is reusable for recovery**: The module-level mock that sets ACTIVE + writes TENANT_STARTED works for both `/start` initial activation and `monitorTenantHealth`'s recovery path, since both expect the same outcome.
- **mockNow = 28_000_000** for TC-028, continuing the per-file convention (+1M from TC-027's 27M).
