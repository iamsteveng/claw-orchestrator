<learnings>
- **Pass `dockerClient` explicitly to `buildApp`**: The `/start` endpoint uses dynamic `import('@claw/docker-client')` when no dockerClient option is passed. This can fail to pick up `vi.mock` in some Vitest configs. Passing a mock docker client directly via `buildApp(prisma, { dockerClient: mockDc })` is always reliable.
- **pollUntilHealthy mock can write audit events**: The mock receives the real `prisma` as its first argument. The implementation can call `prisma.tenant.update()` + `prisma.auditLog.create()` to simulate TENANT_STARTED without real health polling.
- **Flush async void calls with `setImmediate`**: After `/start` triggers `void pollUntilHealthy(...)`, two `await new Promise(resolve => setImmediate(resolve))` calls flush the microtask queue before asserting the TENANT_STARTED event.
- **Mock global fetch for message delivery**: `vi.spyOn(globalThis, 'fetch').mockResolvedValueOnce(new Response(JSON.stringify({ok:true}), {status:200}))` before inject; `fetchSpy.mockRestore()` after.
- **TENANT_DELETED actor is hardcoded `'admin'`** in the DELETE endpoint, not `'system'`.
- **Allowlist API uses snake_case**: POST /v1/admin/allowlist body is `{ slack_team_id, slack_user_id, added_by }`.
- **mockNow = 15_000_000** for TC-015 (continuing from TC-014's 14M baseline).
