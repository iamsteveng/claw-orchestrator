<learnings>
- **Mock openclaw must drain stdin before exiting:** message-server.js writes the message text to `child.stdin` then calls `child.stdin.end()`. If the mock exits before reading stdin (e.g. just `echo`), Node.js emits unhandled EPIPE on `child.stdin` → server process crashes → "socket hang up" in test client. Fix: use `cat > /dev/null` before echoing so the mock drains stdin first.
- **PATH injection for mock executables in spawned servers:** Prepend a tmpDir containing the mock binary to PATH in the server's env. Since message-server.js passes `env: { ...process.env }` to its child spawns, the modified PATH propagates correctly.
- **waitForServer using 401 as readiness signal:** When the server enforces auth, a no-token POST getting 401 (not ECONNREFUSED) reliably confirms the server is up and accepting connections.
