- Relay token validation fires before allowlist check — wrong/missing `x-relay-token` returns 401 before other guards (403, 507, 503).
- To test successful message delivery (200), mock `globalThis.fetch` via `vi.spyOn(globalThis, 'fetch').mockResolvedValueOnce(new Response(...))` — the control-plane forwards to the container via fetch, which would fail without a real container. Always call `fetchSpy.mockRestore()` after the request to avoid polluting other tests.
- The relay token returned in the `POST /v1/tenants/provision` response (`body.relayToken`) is the same value that must be sent as the `x-relay-token` header on message requests.
