<learnings>
- **pollUntilHealthy mock via vi.mock**: Mocking `../../apps/control-plane/src/health-poll.js` with an implementation that directly writes `status=ACTIVE` + `TENANT_STARTED` audit log is the cleanest way to test the state machine without real Docker/network. Two `setImmediate` flushes after POST /start are sufficient for the background `void pollUntilHealthy(...)` to complete.
- **STOPPED → `docker.start` (not `docker.run`)**: For tenants with `previousStatus !== NEW`, the start endpoint calls `dc.start(containerName)` (restart existing container). Only `NEW` tenants use `dc.run()`.
- **Idempotency response code**: Second POST /start on an ACTIVE tenant returns HTTP **200** (not 202) with `{status: 'active'}` — the endpoint special-cases ACTIVE before acquiring the startup lock.
- **`principal` is required in tenant create**: `prisma.tenant.create()` requires the `principal` field (`slackTeamId:slackUserId` format) — omitting it causes a Prisma validation error.
- **`mockNow` starting value**: Use a value distinct from other test files (e.g. 3_000_000 or 10_000_000) to avoid Int32 range issues when many DB operations increment the counter.
