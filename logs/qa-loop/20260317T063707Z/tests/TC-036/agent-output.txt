<learnings>
**pnpm sub-package execa mock path**: When `@claw/docker-client` has its own `node_modules/execa` (not hoisted to the workspace root by pnpm), `vi.mock('execa', factory)` from a root-level test file does NOT intercept the execa used by the docker-client source. The fix is to mock the concrete resolved path: `vi.mock('../../packages/docker-client/node_modules/execa/index.js', factory)` and import from the same path for `vi.mocked()`. This matches the module identity vitest uses in its registry. Tests co-located within the package (like `packages/docker-client/src/index.test.ts`) work fine because mock and source share the same pnpm package resolution context.
