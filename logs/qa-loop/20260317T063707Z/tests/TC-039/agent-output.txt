<learnings>
- **Mock openclaw must drain stdin:** message-server.js writes text to `child.stdin` then calls `end()`. If the mock script exits before reading stdin (e.g. just `echo`), Node.js emits unhandled EPIPE on `child.stdin` → server crashes → "socket hang up". Fix: `cat > /dev/null` before echoing drains stdin so the write succeeds.
- **PATH injection for mock executables:** Prepend a tmpDir to PATH in the server's spawn env. Since message-server.js passes `env: { ...process.env }` to spawned children, the modified PATH propagates to the openclaw spawn.
- **waitForServer using 401 as readiness signal:** When auth is enforced, a no-token request getting 401 (not ECONNREFUSED) is a reliable signal that the server is up.
